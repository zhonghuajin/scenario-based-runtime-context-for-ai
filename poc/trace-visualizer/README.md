# Trace Visualizer

**Trace Visualizer** is a crucial Application Layer component within the [Scenario-Driven Code Context Generator for AI](../README.md) toolchain. Its primary design goal is to assist **human developers** by translating complex runtime logs into intuitive, easy-to-read function call graphs.

While the AI handles the zero-noise, structured code context (e.g., `final-output-combined.md`), the Trace Visualizer helps human developers quickly grasp the execution flow of a specific scenario at a macro level, enabling highly efficient human-AI collaborative development and debugging.

## ✨ Core Features

*   **Parse Call Tree Data**: Directly reads the `final-output-calltree.md` (or related log files) generated by the data processing layer as its input.
*   **Generate Human-Readable Call Graphs**: Renders monotonous log text and Block ID sequences into clear, hierarchical, and visual function call relationship graphs.
*   **Assist in Understanding Execution Flow**: In multi-threaded and complex scenario-driven tests, it helps developers visually track the exact execution path from "Stimulus" to "Response".

## 🏗️ Position in the Architecture

The Trace Visualizer sits at the very end of the pipeline (Application Layer).

1.  **Data Collection & Noise Reduction**: Underlying tools (like `instrumentor` and `block-pruner`) collect and prune zero-noise runtime data.
2.  **Data Structuring**: Generates structured files like `final-output-calltree.md`.
3.  **Visual Rendering (This Module)**: The Trace Visualizer consumes this structured data to render graphical interfaces or charts for human review.

## 🚀 Why is it needed?

Although the main purpose of this project is to provide zero-noise context for Large Language Models (LLMs), **human oversight remains indispensable** in real-world engineering. 

While the AI performs logical reasoning and code review at a micro level based on `Happens-Before` rules, the Trace Visualizer allows developers to "see" how the code runs at a macro level. This combination of "AI reading code, humans reading charts" significantly reduces the cognitive load required to understand complex multi-threaded system behaviors.

## 📦 Getting Started

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed on your machine.

### Installation
Navigate to the `trace-visualizer` directory and install the dependencies using npm (or yarn/pnpm):

```bash
npm install
```

### Running the Application

To start the development server with Hot Module Replacement (HMR):
```bash
npm run dev
```
*This will start the Vite development server. Open the provided local URL (usually `http://localhost:5173`) in your browser.*

### Building for Production

To build the application for production:
```bash
npm run build
```

To preview the production build locally:
```bash
npm run preview
```

## 📖 Prerequisites for Data
Before using the Trace Visualizer, ensure that the preceding steps of the pipeline have been executed successfully and the `final-output-calltree.md` file (containing the hierarchical execution flow and Block ID sequences) has been generated in your workspace.
